# Lexer and Parser

Mi Compilador uses a two-phase approach to process source code: lexical analysis (tokenization) and syntactic analysis (parsing). Both components are automatically generated by ANTLR4 from the grammar definition.

## The Processing Pipeline

```text
Source Code → Lexer → Token Stream → Parser → AST → Visitor → Execution
```

The compiler processes source code through several stages:
1. **Lexer** - Converts source text into tokens
2. **Parser** - Builds an Abstract Syntax Tree from tokens
3. **Visitor** - Traverses the AST and executes the program

## Lexical Analysis (Lexer)

The lexer (`ExpresionesLexer.py`) reads source code and converts it into tokens.

### What is a Token?

A token is a categorized piece of text:

| Token Type | Example | Description |
|------------|---------|-------------|
| `'program'` | `program` | Keyword |
| `'int'` | `int` | Type keyword |
| `ID` | `x`, `result` | Identifier |
| `NUM` | `42`, `100` | Number literal |
| `SUM` | `+` | Addition operator |
| `MUL` | `*` | Multiplication operator |
| `'='` | `=` | Assignment |
| `';'` | `;` | Statement terminator |

### Tokenization Example

Source code:
```text
int x;
x = 10 + 5;
```

Token stream:
```text
'int' ID(x) ';' ID(x) '=' NUM(10) SUM NUM(5) ';' EOF
```

### Generated Lexer

ANTLR4 generates `ExpresionesLexer.py` from the token definitions in the grammar:


## Syntactic Analysis (Parser)

The parser (`ExpresionesParser.py`) takes the token stream and builds an Abstract Syntax Tree (AST).

### What is an AST?

An Abstract Syntax Tree represents the grammatical structure of the program.

For this code:
```text
x = 2 + 3;
```

The AST structure:
```text
Asignacion
├── ID: x
└── Expr (Aritmetica)
    ├── Expr (Numero): 2
    ├── Operator: +
    └── Expr (Numero): 3
```

### Generated Parser

ANTLR4 generates `ExpresionesParser.py` with methods for each grammar rule:

```python ExpresionesParser.py (excerpt)
class ExpresionesParser(Parser):
    def root(self):
        # Parse the root rule: 'program' '{' instrucciones+ '}' EOF
        ...
    
    def instrucciones(self):
        # Parse instructions: declarations, assignments, if statements
        ...
    
    def expr(self):
        # Parse expressions with operator precedence
        ...
```

### Parser Context Objects

Each grammar rule produces a context object containing:
- **Child nodes**: Sub-expressions and nested rules
- **Token references**: Direct access to terminals
- **Helper methods**: `getText()`, `getChild()`, etc.

Example:
```python
class AsignacionContext(ParserRuleContext):
    def ID(self):      # Returns the identifier token
    def expr(self):    # Returns the expression context
```

## The Parsing Process in Prueba.py

The main program (`Prueba.py`) orchestrates the lexer and parser:

```python Prueba.py lines
import sys
import os
from antlr4 import *
from ExpresionesLexer import ExpresionesLexer
from ExpresionesParser import ExpresionesParser
from Visitor import Visitor

def main():
    # Step 1: Read source file
    archivo_ruta = sys.argv[1]
    input_stream = FileStream(archivo_ruta, encoding='utf-8')
    
    # Step 2: Lexical analysis
    lexer = ExpresionesLexer(input_stream)
    token_stream = CommonTokenStream(lexer)
    
    # Step 3: Syntactic analysis
    parser = ExpresionesParser(token_stream)
    tree = parser.root()  # Parse from the 'root' rule
    
    # Step 4: Check for syntax errors
    if parser.getNumberOfSyntaxErrors() > 0:
        print(f"[ERROR] Syntax errors found")
    else:
        print(f"[OK] Parsing successful")
        
        # Step 5: Execute using visitor
        visitor = Visitor()
        visitor.visit(tree)
```

## Error Detection

### Syntax Error Detection

```python
if parser.getNumberOfSyntaxErrors() > 0:
    print(f"[ERROR] El archivo tiene errores gramaticales.")
```

Example errors:
- Missing semicolons
- Unmatched parentheses
- Invalid token sequences
- Unexpected end of file

### Error Examples

<Accordion title="Missing Semicolon">
```text
program {
    int x
    x = 5;  // Error: previous line missing semicolon
}
```
</Accordion>

<Accordion title="Invalid Token">
```text
program {
    int x;
    x = 10 & 5;  // Error: '&' is not a valid token
}
```
</Accordion>

<Accordion title="Unmatched Braces">
```text
program {
    int x;
    x = 10;
// Error: missing closing brace
```
</Accordion>

## Regenerating the Parser

When you modify `Expresiones.g`, regenerate the lexer and parser:

```bash
antlr4 -Dlanguage=Python3 -visitor Expresiones.g
```

This generates:
- `ExpresionesLexer.py` - Tokenizer
- `ExpresionesParser.py` - Parser
- `ExpresionesVisitor.py` - Base visitor class
- `Expresiones.tokens` - Token type definitions
- `ExpresionesLexer.tokens` - Lexer token definitions

<Warning>
Always regenerate after grammar changes. The generated files must match the grammar exactly.
</Warning>

## How ANTLR Handles Precedence

ANTLR parses expressions according to rule order:

```antlr
expr: expr (MUL | DIV) expr    # Higher precedence
    | expr (SUM | RES) expr    # Lower precedence
```

For `2 + 3 * 4`, the parser:
1. Recognizes `3 * 4` as a multiplication expression
2. Treats the result as a single operand for addition
3. Produces: `2 + (3 * 4)`

## Debugging the Parser

To visualize the parse tree:

```python
from antlr4.tree.Trees import Trees

# After parsing
tree = parser.root()
print(Trees.toStringTree(tree, None, parser))
```

<Tip>
Use ANTLR's built-in tree visualization tools during development to understand how your grammar parses different inputs.
</Tip>

## Key Takeaways

- **Lexer**: Converts source text into tokens
- **Parser**: Builds an AST from tokens
- **Context objects**: Represent parsed rules and provide access to children
- **Error detection**: `getNumberOfSyntaxErrors()` checks for syntax issues
- **Auto-generation**: ANTLR generates all lexer/parser code from the grammar
