## Overview

The grammar file `Expresiones.g` defines the syntax of Mi Compilador using ANTLR4. This guide shows you how to extend it with new tokens, rules, and language constructs.

## Understanding the Grammar Structure

The current grammar consists of:

- **Tokens**: Atomic elements like `NUM`, `ID`, `SUM`, `DIV`
- **Parser Rules**: Structural rules like `root`, `instrucciones`, `expr`
- **Labels**: Rule alternatives marked with `#` (e.g., `#Aritmetica`, `#InstrAsig`)

```antlr Expresiones.g (current structure)
grammar Expresiones;

root : 'program' '{' instrucciones+ '}' EOF # Prog ;

instrucciones
    : declaracion ';'                       #InstrDecl
    | asignacion ';'                        #InstrAsig
    | 'if' '(' exprRelacional ')' bloque ('else' bloque)? #InstrIf
    ;
```

## Adding New Tokens

Tokens are defined at the bottom of the grammar file in UPPERCASE.

<Steps>
  <Step title="Define the token">
    Add your token definition after the existing tokens:

    ```antlr
    // Existing tokens
    SUM : '+' ;
    RES : '-' ;
    MUL : '*' ;
    DIV : '/' ;
    
    // New tokens
    MOD : '%' ;       // Modulo operator
    AND : '&&' ;      // Logical AND
    OR  : '||' ;      // Logical OR
    ```
  </Step>

  <Step title="Use the token in parser rules">
    Reference your new token in a parser rule:

    ```antlr
    expr: expr (MUL | DIV | MOD) expr    #Aritmetica
        | expr (SUM | RES) expr          #Aritmetica
        | NUM                            #Numero
        | ID                             #Variable
        | '(' expr ')'                   #Parentesis
        ;
    ```
  </Step>

  <Step title="Regenerate the parser">
    Run the ANTLR4 command to regenerate lexer and parser:

    ```bash
    antlr4 -Dlanguage=Python3 -visitor Expresiones.g
    ```

    This generates:
    - `ExpresionesLexer.py`
    - `ExpresionesParser.py`
    - `ExpresionesVisitor.py`
  </Step>
</Steps>

<Warning>
  Always regenerate the parser files after modifying the grammar. Old generated files will cause runtime errors.
</Warning>

<Steps>

    ```antlr

        | 'while' '(' exprRelacional ')' bloque #InstrWhile
        ;
    ```
  </Step>

  <Step title="Regenerate the parser">
    ```bash
    antlr4 -Dlanguage=Python3 -visitor Expresiones.g
    ```
  </Step>

  <Step title="Implement the visitor method">

  </Step>
</Steps>

### Example 2: Adding String Support

<Steps>
  <Step title="Define string token">
    Add a token for string literals:

    ```antlr
    STRING : '"' (~["\r\n])* '"' ;
    ```
  </Step>

  <Step title="Add string to expressions">
    Modify the `expr` rule:

    ```antlr
    expr: expr (MUL | DIV) expr    #Aritmetica
        | expr (SUM | RES) expr    #Aritmetica
        | NUM                      #Numero
        | STRING                   #Cadena
        | ID                       #Variable
        | '(' expr ')'             #Parentesis
        ;
    ```
  </Step>

  <Step title="Update declaration types">
    Allow string variable declarations:

    ```antlr
    declaracion : tipo ID ;
    tipo : 'int' | 'string' ;
    ```
  </Step>
</Steps>

## Adding New Operators

### Comparison: Adding Modulo (%)

**Before:**
```antlr
expr: expr (MUL | DIV) expr    #Aritmetica
    | expr (SUM | RES) expr    #Aritmetica
    | NUM                      #Numero
    | ID                       #Variable
    | '(' expr ')'             #Parentesis
    ;

MUL : '*' ;
DIV : '/' ;
```

**After:**
```antlr
expr: expr (MUL | DIV | MOD) expr    #Aritmetica
    | expr (SUM | RES) expr          #Aritmetica
    | NUM                            #Numero
    | ID                             #Variable
    | '(' expr ')'                   #Parentesis
    ;

MUL : '*' ;
DIV : '/' ;
MOD : '%' ;  // New modulo operator
```

Then update `visitAritmetica()` in Visitor.py:

```python
def visitAritmetica(self, ctx):
    izq = self.visit(ctx.expr(0))
    der = self.visit(ctx.expr(1))
    op = ctx.getChild(1).getText()
    
    if izq is None or der is None:
        return 0

    if op == '+': return izq + der
    if op == '-': return izq - der
    if op == '*': return izq * der
    if op == '/': return izq // der
    if op == '%': return izq % der  # New modulo operation
    return 0
```

## Best Practices for Grammar Design

<Tip>
  **Operator Precedence**: List operators in order of precedence. Higher precedence operators should appear first:
  
  ```antlr
  expr: expr (MUL | DIV | MOD) expr    // Highest precedence
      | expr (SUM | RES) expr          // Lower precedence
      | NUM | ID | '(' expr ')'
      ;
  ```
</Tip>

### Rule Naming Conventions

- **Parser rules**: Start with lowercase (e.g., `instrucciones`, `expr`)
- **Tokens**: Use UPPERCASE (e.g., `NUM`, `ID`, `MUL`)
- **Labels**: Use PascalCase with `#` (e.g., `#InstrAsig`, `#Aritmetica`)

### Avoid Left Recursion Issues

<Warning>
  ANTLR4 handles direct left recursion but not indirect left recursion.
  
  **Bad (indirect left recursion):**
  ```antlr
  expr : term ;
  term : expr '+' expr ;  // Indirect left recursion through term
  ```
  
  **Good:**
  ```antlr
  expr : expr '+' expr | term ;
  term : NUM | ID ;
  ```
</Warning>

### Keep Rules Simple and Clear

```antlr
// Bad: Complex nested rule
instrucciones : ('int' ID | 'string' ID) '=' expr ';' #Decl ;

// Good: Separate concerns
instrucciones : declaracion ';' #InstrDecl ;
declaracion : tipo ID ('=' expr)? ;
tipo : 'int' | 'string' ;
```

## Common Pitfalls

<Warning>
  **Issue**: Parser doesn't recognize new token
  
  **Solution**: Make sure you defined the token in UPPERCASE and regenerated the parser files.
</Warning>

<Warning>
  **Issue**: Visitor method not called
  
  **Solution**: Check that you added a label (e.g., `#InstrPrint`) to your grammar rule. Only labeled alternatives generate visitor methods.
</Warning>

<Warning>
  **Issue**: Syntax errors with existing programs
  
  **Solution**: Your grammar changes may have broken backward compatibility. Review operator precedence and rule order.
</Warning>
